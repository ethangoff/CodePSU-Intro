Let's go over a few technical items to avoid confusion.

You should have 1 laptop for your team that is ready to go for writing C, C++, or Java.

Your code will be compiled on Linux, but it's just fine to work on Windows or Mac.
We'll only be using standard libraries anyway.

All submissions are subject to a secret time limit; take too long, and your process is killed.
Hint: Efficient code.

If you use C or C++, you must use `int main()` rather than `void main()` and NEVER use `system("pause")`.

If you need `stdlib.h` or `math.h`/`cmath` you MUST explicitly `#include` these even though Visual Studio does not enforce this.

Hint: Just include these header files in every submission.

Do not print to the console to prompt a 'user' for input. Anything your program prints is considered output and will be judged.

Read each problem carefully and pay attention to the given input bounds.

We have two different problem packets. One packet for the advanced tier and one for the intermediate tier. If you have or are taking a 300-level or higher CMPSCI class (sans 360), you must compete in the advanced tier.

In order to be more like the ACM's ICPC, all problems are weighted equally.

The submission system needs the Java JDK installed on your computer to run. It's a big download, so do that now if you haven't already.
